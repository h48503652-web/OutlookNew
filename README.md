# פתיחת טיוטות Outlook מתוך אפליקציית Web

## סקירה
אפליקציית Web פשוטה שמתקשרת עם שירות מקומי ב-Windows כדי לפתוח טיוטות מייל ב-Outlook (אחת לכל נמען) כולל קובץ מצורף. לא מתבצעת שליחה אוטומטית.

## למה לא mailto:
- `mailto:` אינו מאפשר לצרף קובץ מקומי אוטומטית באופן אמין בין דפדפנים.
- מגבלות אורך כתובת URL (נושא/גוף) והצגה יחידה (לא מרובה נמענים לטיוטות נפרדות).

## למה Web לא יכול לפתוח Outlook ישירות או לגשת לקבצים:
- דפדפן מבודד מתהליכי מערכת, אין לו הרשאות לגשת ל- Outlook/COM.
- גישה לקובץ מקומי אפשרית רק דרך `<input type="file">` ומוגבלת; אין נתיב מלא או הרשאות לפתיחה ע"י תוכנה מקומית.

## ארכיטקטורה
- Frontend סטטי: טופס HTML/JS שמעלה `multipart/form-data` ל-`http://127.0.0.1:5005/draft`.
- Helper מקומי (Windows): Flask + pywin32, פותח טיוטות ב-Outlook באמצעות COM, מצרף את הקובץ ושומר זמנית.

```
Browser (Form) ---> POST /draft (localhost) ---> Flask ---> Outlook COM ---> Draft windows
```

## התקנה והרצה (Windows)
1. ודאי ש-Outlook מותקן ומחובר לחשבון פעיל.
2. התקיני Python 3.10+ (עם "Add Python to PATH").
3. פתחי טרמינל PowerShell בתיקיית `helper` והפעילי סביבה:
   ```powershell
   python -m venv .venv
   .venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   python app.py
   ```
   יופיע שרת על `http://127.0.0.1:5005`.
4. פתחי את הקובץ `web/index.html` בדפדפן (אפשר ב-"Open With Live Server" או כפול-קליק).
5. מלאי את הטופס, צרפי קובץ, ולחצי "פתח טיוטות". טיוטות Outlook ייפתחו.

## הערות אבטחה
- השירות מאזין רק ל-`127.0.0.1`.
- הקובץ נשמר בתיקייה זמנית ומוסר לאחר הפתיחה.

## שיפורים אפשריים
- חתימה/תבניות HTML.
- אימות כתובות מייל ו-CC/BCC.
- הגדרות פורט ואבטחה בסיסית (API key מקומי).
- בנדלר סטטי ושיפור UI.
